# 编码实践（通用）

> 本文件只保留跨语言、可判断的决策要点。语言具体写法、禁止项与库选型见同目录分语言文件；目标性表述见 [`main.md`](../main.md)。
>
> 下列章节与分语言文件一一对应；无跨语言判据的节作占位，细节落在分语言文件。

## 0. 语言版本与语法

- **新语法以可读性为准绳**：新写法更难懂时（深度嵌套 `:=`、把简单分支硬写成 `match`、过度类型体操）选直观写法，不为「显得现代」而用。

## 1. 命名与代码风格

- **格式不进 review**：缩进、引号、分号、import 排序全部交给格式化工具，review 中不讨论格式。

## 2. 函数与模块设计

- **拆分信号**：出现「阶段性注释」（`// 第一步…`）、超过一屏、或一个函数名需要用「and」才能描述时 → 拆成更小的具名函数。
- **参数阈值**：参数超过 3 个 → 改具名对象/参数（具体写法见分语言文件）；避免布尔陷阱参数，用枚举或拆分函数替代。

## 3. 类型安全与错误处理

- **错误分流判据**：业务上**可预期**的失败（找不到、校验失败、外部调用失败）→ 用返回值表达；**不可恢复**的异常情况 → 才用异常/panic。
- **捕获即处理**：捕获后必须做有意义的事（恢复 / 转换 / 上报 / 带上下文重抛）；禁止空 catch、禁止「只打日志后继续」。
- **重抛保根因**：包装错误时必须携带原始 cause，不丢失定位信息。

## 4. 异步与并发

- **并发模型选择**：IO 密集 → 异步/goroutine 并发；CPU 密集 → 多进程 / worker / 线程池，不阻塞事件循环或主线程。
- **生命周期闭环**：每个并发单元都有明确退出条件与错误处理，不留悬空任务/泄漏 goroutine；批量并发必须限流。

## 5. 性能与优化

- **先测后调**：优化前用 profiler/benchmark 定位真实热点，优化前后对比并纳入回归，不靠直觉做过早优化。
- **常见浪费清单**：N+1 查询、循环内重复 IO/计算、不必要深拷贝、无索引查询 → 优先批量与缓存；缓存必须有明确失效策略。

## 6. 注释与文档

- **写 why 不写 what**：注释解释意图、权衡、坑与非显而易见的约束，不复述代码字面逻辑；禁止 `i++ // 自增` 这类废话注释。
- **同步与清理**：改代码同步更新注释；删被注释掉的死代码（靠版本控制找回）；`TODO`/`FIXME`/`HACK` 必须附负责人或 issue 链接。

## 7. 测试规范

- **测行为非实现**：针对公共接口与可观察行为断言，不断言私有细节，降低重构脆性。
- **隔离确定**：单测不依赖网络/真实 DB/时钟/全局状态；外部依赖 mock/fake，时间用可注入时钟。
- **覆盖优先级**：先覆盖核心逻辑、分支与边界（空、极值、错误路径），覆盖率作参考非目标；修 bug 先写能复现的失败用例再修。

## 8. 安全编码

- **边界校验**：所有外部输入先校验再用，按输出上下文（HTML/SQL/Shell/URL）转义。
- **杜绝注入**：SQL 用参数化/ORM，子进程传参数数组，禁止字符串拼接。
- **加密选型**：密码用 `argon2`/`bcrypt`（不用 MD5/SHA1）；随机数用密码学安全源，不用通用随机数 API。
